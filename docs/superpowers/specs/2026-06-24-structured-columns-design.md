# 主表结构化多列输出 — 设计文档（iteration-3）

- 日期：2026-06-24
- 状态：草案（brainstorming 产出，待 grilling 复核）
- 范围：主表（分层版 + 扁平版）行尾列结构化；不动匹配逻辑、不动边界表。
- 依赖：iteration-2 产出（匹配日志字符串已含全部信息）。

## 1. 背景与目标

当前主表行尾只有单一「匹配日志」列，塞了**多类信息**（匹配阶段 + 粗筛子类型 + 单年说明 + 选科漂移 + 复核结果 + 原因细节）。7091 行含 ≥2 类标记，无法直接按「匹配阶段」或「单年」筛选。

**目标**：把单一日志列**拆成 5 个结构化列**，每列只放一类信息，便于筛选（如筛"严格匹配+单年数据"的行）。

## 2. 设计（grilling 确认）

### 2.1 五个新列（替换原「匹配日志」单列）

| 列名 | 取值 | 来源（解析自原日志） |
|------|------|------|
| **匹配阶段** | `严格匹配` / `粗筛匹配` / `语义匹配` / `新增专业` / `特殊情况` / `复核存疑` / `疑似改名校` / `新校无历史` | 日志前缀 |
| **单年数据** | `是` / (空) | 日志含「（单年数据，无标准差）」 |
| **选科漂移** | `是` / (空) | 日志含「选科政策漂移，已忽略」 |
| **复核结果** | `确定` / (空) | 判断型匹配（粗筛+语义）经二次复核通过；严格/新增/特殊/改名 留空 |
| **原因备注** | 自由文本 | 剩余细节（如「归一化专业名+招生类别一致」/「核心名唯一」/「括号子集消歧（…）」/语义理由/复核存疑原因/估算口径） |

- flag 格式：**是/空**（满足填「是」，不满足留空；筛选时"非空"即满足）。
- 粗筛子类型（核心名唯一 / 括号子集消歧）**归入「原因备注」**（不单列，避免列过多）。

### 2.2 主表行尾列顺序（新）

大绿本原 12 列 + 行尾追加 **7 列**（原 J/T + 新 5 列，原「匹配日志」移除）：

```
…(原12列) | 近三年统计线差 | 近三年线差标准差 | 匹配阶段 | 单年数据 | 选科漂移 | 复核结果 | 原因备注
```

- 分层版 + 扁平版均应用（仅主表）。
- 边界表（新增/被删/特殊/改名/新增校/停招）**保留原单一日志列**，不拆（它们的日志简单，且用途是单列待人工核验）。

### 2.3 解析规则（纯函数）

从现有匹配日志字符串解析出 5 列（不动匹配逻辑，只改输出呈现）：

```
匹配阶段 ← 日志前缀（严格匹配/粗筛匹配/语义匹配/新增专业/特殊情况/疑似改名校/新校无历史；复核存疑→特殊情况或独立？见 §3）
单年数据 ← "（单年数据" in 日志 ? "是" : ""
选科漂移 ← "选科政策漂移" in 日志 ? "是" : ""
复核结果 ← (匹配阶段∈{粗筛匹配,语义匹配}) ? "确定" : ""   # 主表判断型皆经复核确定
原因备注 ← 去掉前缀+flag标记后的剩余文本（如"核心名唯一"/"括号子集消歧（…）"/估算口径/存疑原因）
```

- 复核存疑行已在特殊表（不在主表），故主表无"复核存疑"阶段；若分层版标注专科/特殊行的日志也按此解析（匹配阶段=对应类别）。

## 3. 已定细节

- 复核存疑（已移特殊表）的日志**不拆**（边界表保留原日志列，§2.2）。
- 原因备注**去掉前缀**（如"严格匹配："已体现在「匹配阶段」列，备注只留剩余细节，不重复）。
- 分层版专科行的「专科：不在本次整理范围」→ 匹配阶段列填「专科（超范围）」，原因备注空。
