# 山东省高考录取数据匹配整理 — 设计文档（Spec）

- 日期：2026-06-23
- 状态：草案（brainstorming 产出，待 grilling 细化）
- 范围：**纯数据管线**，无 UI / 后端（baoyu-design、visual-companion 不适用）
- 目的：把 2026 大绿本招生计划（主键）与近三年录取线差（外键）按专业名语义一一对应，输出带线差与日志的整理表，并沉淀为可年度复用的 agent skill。

## 1. 背景与目标

2026 高考已结束、6/25 出分。为辅助志愿填报，需把今年的招生计划（大绿本）与过去三年各专业录取线差关联起来。因每年招生计划会增删/改名专业，无法直接按字符串对应——需**语义匹配**。

核心目标：以大绿本专业名为**主键**，在合并后的近三年数据中找**唯一**对应专业，在大绿本每个专业行尾追加「近三年统计线差(J)」「近三年线差标准差(T)」「匹配日志」三列。保证：近三年数据中不存在语义歧义，每个大绿本专业至多对应 1 个。

## 2. 数据资产（实证结构，2026-06-23）

### 2.1 大绿本 `山东省2026年大绿本招生计划.xlsx`（主键源）
- 单 sheet，27610 数据行，12 列。**分层结构**：批次头行 / 小标题行 / 学校行 / 专业行。
- 列：A 批次 · B 小标题 · C 学校代码 · D 学校名 · E 代号 · **F 名称（专业名，匹配键）** · G 选考科目要求 · H 学制 · I 计划数 · J 学校备注 · K 年收费 · L 整行校准。
- **专业行判别**：E(代号) 与 F(名称) 同时非空。
- 批次(A) 取值：`4.常规批`(25591) · `1.提前批A类`(1249) · `2.提前批B类`(765) · `3.提前批—飞行技术(军队)`(5)。
- 专业行总数 25655；按批次：常规批 23887 / 提前批A类 1139 / 提前批B类 627 / 飞行技术 2。
- 小标题(B) 17 类，含 普通计划(23114)、中外合作办学(1551)、高校专项计划(613)、军事类(487)、公安政法类(409)、定向培养军士生(专科)(229，含非专业行；其中专业行 181 见 §3) 等。

### 2.2 近三年 `近三年学校批次专业线差统计.xlsx`（常规批一段 J/T 源 + 部分 提前批）
- sheet `统计结果`：46648 行，29 列。关键列：A batch · B schoolcode · C schoolname · D majorname · E subject · F remarks · **G 基础专业名** · **H 是否括号专业** · **I 括号内容** · **J 统计线差** · K-M 2023/2024/2025线差 · … · **T 线差标准差** · … · Y 同基础专业版本数 · AC 同基础专业版本摘要。
- batch 取值：`常规批一段线`(28269) · `常规批二段线`(17554，**删除**) · `提前批`(825，**弃用**，见 §3)。
- 辅助 sheet：`括号语义判断`(7928)、`同基础专业多版本`(9209)、`较2025浮动提醒`、`说明`。
- `说明` 记录口径：线差=`low_score − 山东一段线`，2023=443 / 2024=444 / 2025=441；预测=可用年份线差均值（单年直接用该年）。

### 2.3 提前批补充表 `山东省高考提前批录取数据.xlsx`（提前批 J/T 计算源）
- 单 sheet，1900 行，43 列。列：A 批次名称 · B 招生类别 · C 院校代码 · D 院校名称 · E 专业代码 · F 专业名称 · G 选科 · H-S 三年录取{人数/高分/均分/低分} · …
- **无现成线差**，仅有原始分。批次名称：`本科提前批A类`(1161) · `本科提前批B类`(546) · `专科提前批`(193，**删除**)。
- 本科 A+B 共 1707 行；每行拥有「录取低分」的年份数分布：3 年全有 866 / 2 年 254 / 1 年 585 / 全无 2。

## 3. 锁定的关键决策

| 项 | 决策 | 依据 |
|----|------|------|
| 范围 | 仅 提前批 + 常规批一段，**仅本科** | 用户 |
| 提前批 AB 类 | **无差别**，匹配/合并时合并为同一组 | 用户（AB 仅为填报观察便利） |
| 提前批数据来源 | **以补充表为准**；近三年里 825 条 `提前批` 弃用 | 用户（补充表是漏加的完整数据） |
| 提前批线差 | 自算 `录取低分 − 当年一段线`（443/444/441）；统计线差=可用年份均值；标准差同口径 | 用户（与近三年一致） |
| 专科排除 | 全排除 | 用户 |
| 大绿本专科判别 | **小标题(B) 含「专科」** → 181 行（全是 提前批B类 的 定向培养军士生(专科)） | 实证：信号B=181 是校名信号(140)的超集 |
| 近三年专科 | 删 `常规批二段线`(含专科校)后，`常规批一段线` 已 0 专科校名，天然纯本科 | 实证 |
| 源文件 | **只读，永不修改** | 用户 |
| 源文件位置 | 移到 `data/`（git mv，保留历史） | 用户 |

> 弃用 825 提前批前需先做**重叠验证**（脚本，只读）：确认这 825 条的 `(schoolcode, 归一化majorname)` 均已被补充表覆盖，无独有数据；若有独有行，提取单列交人工核验，不静默丢弃。

## 4. 合并后的数据模型

### 4.1 统一历史表（Stage 0 产出，匹配的右侧/外键池）
来源：
- 近三年 `常规批一段线`：28269 行（J/T 已算好）。
- 补充表 `本科提前批A+B`：1707 行（AB 合并，batch 统一记 `提前批`，**现场计算 J/T**）。
- 合计 ≈ **29976 行**。

Schema（对齐近三年）：`batch, schoolcode, schoolname, majorname, 基础专业名, 是否括号专业, 括号内容, subject, 统计线差, 线差标准差, 可用年份数, 来源表`。

提前批 J/T 计算式：
- `year线差 = 录取低分 − 当年一段线`（2023=443/2024=444/2025=441）。
- `统计线差 = mean(有数据的年份线差)`；单年直接用该年。
- `线差标准差 = std(有数据的年份线差)`；单年为空/0。

### 4.2 大绿本本科专业表（Stage 0 产出，匹配的左侧/主键池）
来源：大绿本专业行，排除小标题含「专科」(181)。
- 常规批 23887 ＋ 提前批(A类 1139 ＋ B类本科 446) ＋ 飞行技术 2 ≈ **25474 行**。
- 保留原批次标签用于回填，但提前批 A/B 在匹配池中合并。

## 5. 匹配键规则（核心）

### 5.1 归一化（用于严格匹配）
- 全角↔半角标点统一（大绿本用半角 `,`，补充表用全角 `，`；括号 `（）`↔`()`）。
- 去首尾空格、合并连续空白。
- 校名同样归一化（近三年有 `三亚学院(中外合作办学) ` 带尾空格、并把合作信息挂校名的情况）。

### 5.2 括号四「区分类」必须对齐（同名却不同专业）
1. 专业方向/描述（如 `量化投资`、`含电力储能方向`）
2. 性别（`男` / `女`）——**同名按性别拆成两条独立匹配**（不同分数线）
3. 校外合作（`中外合作办学` / `校企合作` / 中外合作国别）
4. 独立校区

### 5.3 「忽略类」括号内容（不影响是否同一专业）
身高 / 体重 / 色盲 / 色弱 / 视力 / 年龄 / 单科成绩 / 体检标准 / `特殊类型招生控制线` 等考生客观要求——即使两年要求不同也不算不同专业。性别是唯一例外（见 5.2）。

### 5.4 辅助字段
优先利用近三年的 `基础专业名`(G) 与 `括号内容`(I) 列做归一化与区分类识别；`括号语义判断` sheet 可作为参考。

## 6. 管线设计（方案 A：脚本严格初筛 → agent 语义 → 边界三表）

### Stage 0 — 预处理合并（脚本，只读源，不改源文件）
- 删 `常规批二段线`；弃 825 `提前批`（先做 §3 重叠验证）。
- 补充表算线差并入，AB 合并、删专科提前批。
- 大绿本排除小标题含「专科」；导出本科专业表。
- 产出统一历史表 + 大绿本本科专业表（落 `intermediate/`）。

### Stage 1 — 严格匹配（脚本）
- 键 = `(归一化校名, 归一化专业名)`，在统一历史表中精确查找。
- 命中即取 J/T，日志 `严格匹配：归一化专业名一致`。
- 预期覆盖大头（多数专业名未改名）。

### Stage 2 — 语义匹配（**agent，禁脚本**）
- 对 Stage 1 未命中的大绿本专业，**并行 agent** 逐个推理。
- 每个 agent 输入：大绿本未命中专业（校名+全名+选科+小标题）＋其同校的历史候选集（按 `基础专业名` 预筛）。
- agent 输出：唯一对应的历史行（或「无对应」）＋ J/T ＋ 简短日志（写明区分类对齐理由）。
- 因近三年无语义歧义，每个至多 1 个对应。

### Stage 3 — 边界处理
- **新增专业**（大绿本有、历史无）：`估算统计线差 = 同校 ＋ 同选科 的历史专业 统计线差 均值`；日志 `新增专业：估算=同校同选科(<n>)均值=<值>`；行加特殊标记并单列成表。
- **被删旧专业**（历史有、大绿本本科表无）：单列成表，日志 `近三年有、2026 大绿本无`。
- **特殊/无法处理**：单列成表，日志写明原因。

## 7. 输出产物（`output/`）
- `大绿本_附线差.xlsx`：**完整复制大绿本**（保留分层结构：批次头/小标题/学校行）＋ 行尾追加 3 列 `近三年统计线差` / `近三年线差标准差` / `匹配日志`（仅专业行填值）。**不覆盖**大绿本原列（含原 J 学校备注）。
- `新增专业.xlsx` / `被删旧专业.xlsx` / `特殊情况.xlsx`（各带日志列）。
- `README.md` 文档化全过程。

## 9. 日志列规约（简短备注，便于人工核验）
- 严格匹配：`严格匹配：归一化专业名一致`
- 语义匹配：`语义匹配：<区分类对齐理由，如 括号方向'量化投资'对应>`
- 新增专业：`新增专业：估算=同校同选科(<n>)均值=<值>`
- 被删旧专业：`近三年有、2026 大绿本无`
- 特殊：`无法匹配：<原因>`
- 弃用行（825 验证用）：`近三年提前批，已被补充表覆盖` / `独有，待人工核验`

## 11. 待 grilling / prototype 确认的开放点
- 归一化算法精确化（全角/半角映射、空白）→ prototype 验证命中率。
- `subject` 选科匹配粒度：历史 `subject` 形如 `物理和化学` 或多选 `物理 | 物理和化学`；新增专业「同校同选科」如何判定等价。
- 大绿本「飞行技术(军队)」仅 2 行：是否单独处理或并入提前批。
- 输出是否严格保留大绿本分层结构（vs 仅专业行扁平表）——默认保留分层。
- 语义匹配 agent 的批大小/并发/prompt 模板 → 实现计划阶段。
- 一段线对提前批是否始终适用（已确认用一段线，grilling 复核）。

## 12. 验收标准
- 三源文件未被修改（字节级/哈希比对）。
- 大绿本每个本科专业行均有匹配日志；匹配或边界归类覆盖 100%。
- 提前批线差口径与近三年一致（低分−一段线）。
- 新增专业估算公式正确且带标记；三张边界表齐全且带日志。
- 专科 181 行（小标题含「专科」）全部排除；常规批两侧无专科残留。
- 可复用 skill 产出并能独立运行。
