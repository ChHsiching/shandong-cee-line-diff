# 山东省高考录取数据匹配整理

把**今年的招生计划**（大绿本）里每个专业，和**往年的录取数据**按专业名语义匹配，给每个专业补上「往年线差」和「线差标准差」。匹配不上的单独列出，并标注原因。

匹配分两步：**程序严格匹配**（归一化后专业全名完全一致 = 直接匹配）+ **agent 语义匹配**（程序没匹配上的，由 agent 逐条读专业名、理解语义后判断能不能和往年某个专业配上）。agent 匹配完，再做一轮 **agent 二次复核**（逐条验证每个 agent 判断的匹配是否正确），确保结果准确。

## 第一步：问清楚数据在哪、怎么算

收到数据文件后（或用户提到山东高考/大绿本/线差），用 **AskUserQuestion** 逐批问以下问题。**用户在调用 prompt 里已经说了的就不问，只问没说的。** 能从文件内容看出来的（如年份、列名）也不问，直接读。

### 第 1 批（文件位置 + 一段线）

1. 今年的招生计划（大绿本）xlsx 文件在哪个路径？
2. 往年的录取数据 xlsx 文件在哪个路径？
3. 有没有补充录取数据（主统计表里没统计到、只有录取分、需要现场算线差的）？**可能有 0 个、1 个或多个，不一定是提前批**（今年可能是提前批，明年可能是别的批次/类型）。如果有，每个都要问清楚：文件在哪、是什么批次/类型、线差从哪一列算（录取最低分？平均分？）。
4. 往年每年的「一段线」（本科录取控制分数线）分别是多少？（如果数据文件里已经写明了就不用问）

### 第 2 批（计算方式 + 精度）

5. 往年数据里有没有现成的「线差」和「线差标准差」？还是需要从录取分数自己算？
6. 如果需要自己算：线差 = 录取最低分 − 一段线，对吗？还是有别的算法？
7. 计算结果保留几位小数？（默认 2 位）
8. 输出表需要在原始数据后面加哪些列？（默认：线差、标准差、匹配结果、原因说明）

### 第 3 批（范围 + 特殊处理）

9. 只整理本科？还是也包含专科？只常规批？还是也包含提前批？
10. 没匹配上的专业怎么处理？（估算一个参考值 / 留空 / 单独成表 / 以上都要）
11. 有没有已知改名的学校？如果有请列出来。
12. 输出格式：保留原始大绿本的层级结构（批次标题、学校行）？还是只要扁平的专业行？

**问完才开始处理。不确定的继续问，不猜、不静默跳过。**

问到的文件名 / 一段线 / 补充表配置，跑 `run_pipeline` 时通过 CLI 参数传（`--dl-file` / `--one-line` / `--supplement-batches` / `--supplement-low-cols` 等，见 REFERENCE 的「run_pipeline CLI 参数」段），**不要硬编码到代码里**——数据年年变，参数化才能复用。

**问卷答完，整个整理流程按 REFERENCE 的「管线串联命令」段（8 步）执行**：跑管线 → 派改名 agent → 跑管线（重生成改名感知 batch）→ 派语义 agent → 跑管线 → 派复核 agent → 跑管线 → 审计。SKILL 第二~七步是每步**在做什么**的逻辑说明（你跑管线时这些自动发生），你只需要按 REFERENCE 的命令串联 + 在「派 agent」步骤派 subagent（并发 ≤6）。

## 第二步：读取数据 + 程序严格匹配

### 2.1 读取源文件

- **大绿本**：今年招生计划，按批次/学校/专业分层。专业行 = 代号和名称都非空。
- **往年录取数据**：往年每个学校每个专业的录取分数、线差、标准差。
- **补充录取数据**（**可选**，可能有 0/1/多个，批次不限）：主统计表里没统计到、只有录取分（最低分等）、需要现场算线差的数据。补充表不一定有、不一定几个、**不一定是提前批**——按第一步问清的实际为准；没有就跳过，主统计表已统计的专业直接用现成线差。

每个文件读取前后都做 SHA256 校验，确保字节没被改过（提前批补充表如果不存在就跳过）。专科行（小标题含「专科」）排除。

### 2.2 程序严格匹配

对大绿本的每个专业，取专业名做归一化（统一全角半角、去空白、剥掉体检/语种/学费等不影响专业身份的描述性括号），然后和往年数据里的专业名精确比对。**归一化后完全一致 = 匹配成功**，直接填入往年的线差和标准差。

这一步是确定性的，不需要 agent 介入。大约能匹配 58%。

**完成判据**：每个大绿本本科专业都被处理过（匹配成功或标记为「待 agent 匹配」）。

## 第三步：agent 语义匹配

程序严格匹配没配上的专业，交给 agent 逐条处理。

### 怎么做

1. 对每个没配上的专业，找出同一所学校、专业名相似的往年候选（核心名 = 去掉所有括号后的基础专业名，核心名相同的就是候选）。
2. **agent 逐条读**：今年这个专业叫什么、往年候选分别叫什么，**语义判断**是不是同一个专业。
3. 语义相同 → 匹配成功，填入对应候选的线差和标准差。
4. 语义不同 → 标记「无法匹配」。
5. 不确定 → 标记存疑，留到第四步问用户（但应该很少，绝大多数 agent 自己就能判断）。

### agent 怎么判断「同一个专业」

单一判据——**基数规则**（往年数据是基准，今年往往年上靠）：

- **一对一、一对多可以；多对一不行。** 往年某校某核心专业只有 1 个 → 今年这个核心下的**所有变体**都匹配这 1 个（**一对多**：1 个往年 ↔ 多个今年）。往年同核心有多个 → 今年每个专业按实际描述对应到其中**某一个**（**一对一**）。绝不能把往年多个并到今年一个上（**多对一**）——多个往年里没对上的该进「停招」表，不能拿它们的均值凑一个线差给今年。

在这个判据下，差异分两类：

- **永远不算身份的差异**（任何时候都当核心名本身看，对得上就匹配）：
  - **培养模式标签**：拔尖、卓越、创新、英才、基地、未来、试验班、订单班等「XX 班」——只是培养组织方式，不创造新专业。往年「数学」对今年「数学(拔尖)」永远没问题，哪怕往年同核心有多个。
  - **描述性噪音**：标点、词序、体检/身高/色盲、学费、语种、章程引用、加减括号内容、子专业清单的大段描述。今年的专业名通常比往年起得长（大绿本写一大段），往年通常就一个短名——看核心名 + 主要方向就够，多余描述不影响。
- **一对多时被吸收、一对一时才算不同专业的差异**：中外合作、师范、性别（男/女）、招生类别（普通/地方专项/综合评价）、真正不同的方向（如「投资学(量化投资)」≠「投资学」）。
  - 往年同核心只 1 个（**一对多**）→ 今年这些变体也都匹配那 1 个（只有这 1 个数据可用）。
  - 往年同核心多个（**一对一**）→ 这些差异就是不同专业了，今年每个专业按描述挑往年**一个**对应的；今年这个专业若往年只有「中外合作」「师范」、今年是「普通」这种对不上的，就别硬配，走第五步估算。

关于学校（匹配的前提是**先同校**）：

- **独立招生的不同校区 = 不同的学校**（华北电力大学(北京) ≠ 华电(保定)）——校名不同，匹配前就按校名分开了，根本不会进同一个候选池。同一学校多校区一起招生的不算不同学校。
- **学校改名/转设/合并** → 上网搜所有消失/新增的学校确认（见 5.2）。
- **校名/校区不确定** → 上网搜招生章程确认。

### 数据量大时

25474 个专业中大约有 5000-6000 个需要 agent 匹配。用 subagent 并行处理（每个 subagent 处理一批，约 20 个专业），可以同时派多个 subagent。**不惜 token，追求准确**：每个专业都要认真读、认真判断，不能偷懒跳过。

**完成判据**：每个待 agent 匹配的专业都有结果（匹配成功 + 对应候选 / 无法匹配 / 存疑）。

## 第四步：agent 二次复核

所有 agent 判断的匹配（程序严格匹配以外的全部），再做一轮独立复核：另一个 agent 逐条看「这个匹配对不对」，判「确认」或「存疑」。

- 确认 → 留在主表。
- 存疑 → 移到特殊情况表，不进主表。

### 复核时怎么判断

复核 agent 用的就是第三步那条**基数规则**（一对一/一对多可以、多对一不行；培养模式标签永远不算身份；中外合作/师范/性别/类别/真方向在一对多时被吸收、一对一时才算不同专业）。每条复核任务的 `requirement` 字段会**按「往年同核心几个」自动给出对应那版的规则**（程序已数好往年同核心数），照着判就行：

- 往年同核心只 1 个 → 今年任何校内变体都判「确认」（一对多）。
- 往年同核心多个 → 培养模式标签差异判「确认」；中外合作/师范/招生类别/真方向实质不同判「存疑」。
- 不确定 → 判「存疑」，宁可保守。

**这一步是硬性的**：主表里不能有未经复核的判断型匹配。宁可保守（多移几条到特殊情况），也不能让错配留在主表。

**完成判据**：每个判断型匹配都有复核结果（确认 / 存疑）。

## 第五步：处理边界情况

### 5.1 agent 确认匹配不到的专业 → 逐级估算

agent 认真读后**确认**匹配不到往年同一个专业（不是「不确定」，是确认不是同一个）→ 按以下顺序逐级估算线差参考值，标注「估算」（和真实匹配的线差分开，不混淆）：

1. **同校 + 同选科**的其他专业线差均值（最精确）。
2. 同校 + 所有专业线差均值（同选科没有时的退化）。
3. 整所学校往年完全没在山东招过生 → 无法估算，线差和标准差留空，标注「新校无历史数据」。

线差和标准差都保留 2 位小数。标准差 = 所用专业的线差标准差的均值（不重新算总体标准差）。

注意：去年今年虽然改了名字/方向但本质是同一个专业的（如「临床医学(5+3一体化儿科学)」vs「临床医学(5+3一体化,儿科学)」逗号差异）→ 这是匹配成功，不进估算。

**「对不上」的也估算（不留空）**：同核心名往年有、但构成**多对一**（如今年 1 个大类 ↔ 往年多个具体专业）或类别/方向全对不上、确认找不到有效对应的——和真新增一样按上面三级估算，结果写进 `未能匹配的专业.xlsx` 的估算列 + 原因说明写清楚「对不上，按同校同选科均值估算」。只有**表意特别模糊**、连选科都看不出门道的，才留空。

### 5.2 改名/转设/合并的学校 → 网查 + 联动

往年有、今年消失的学校 × 今年有、往年没有的学校 → 可能是改名了。

**网查每一所**：对每一所消失或新增的学校都上网搜索，确认是否改名/转设/合并。候选列表给全部历史独有校（不只字面相似的）——真正的旧校名可能字面差异很大（如「佛山科学技术学院」→「佛山大学」），不要因字面不像就放过，必要时直接搜新校名的官方信息确认前身。

确认改名后，程序自动联动：
- 新校名 + 旧校名记入改名表。合并情形（一个新校名由多所旧校合并，如应急管理大学 = 华北科技学院 + 防灾科技学院）每个旧校名都记。
- 旧校名的往年线差数据并入新校名——改名的学校今年招的专业，在程序严格匹配 + agent 语义匹配阶段自动用旧校名的往年数据（不再 J/T 留空）。
- 旧校名从「停招消失校表」移出，新校名从「新增校表」移出（它们是改名的，不是真停招/真新增）。

网查结论 + 官方来源链接（教育部 moe.gov.cn 或省政府公文，**不认**第三方百科/新闻）填进 `rename_result.jsonl` 每行的 **`note`** 字段（is_rename=true 的每行都填）——它会直接进改名表备注列，是用户看到改名依据的唯一入口。网查详情另存 `research/<新校名>.md`（每校一个文件）备查。

### 5.3 被删的旧专业

往年有、今年该校还在但没招这个专业了 → 记录到被删旧专业表（保留往年的线差数据）。

### 5.4 新增的学校

今年有、往年完全没有在山东招生的学校 → 记录到新增校表。

### 5.5 停招消失的学校

往年有、今年完全没在山东招生的学校（排除已确认改名的）→ 记录到停招消失校表。

**完成判据**：每个大绿本本科专业都有明确归宿——匹配成功 / 估算 / 特殊情况 / 改名联动。

### 追加列说明（主表行尾，不覆盖原始数据）

1. **往年线差**：匹配到的专业的往年线差（估算的标注「估算」）
2. **往年线差标准差**：同上
3. **匹配方式**：严格匹配 / agent 语义匹配 / 新增专业估算 / 特殊情况 / 改名 / 新校无历史 —— 用来筛选
4. **原因说明**：用大白话写清楚为什么这么匹配、或者为什么没匹配上（不写术语，写人话）

### 审计

跑完管线后，必须跑审计脚本确认结果正确：

- 每个本科专业都有匹配方式标注（不能空）
- 匹配到的线差和标准差与往年源数据一致
- 没有全空的数据行
- 每张表都有数据
- 所有判断型匹配（agent 匹配的）都经过了二次复核

**审计 exit 0 才算完成。** pytest 全绿 ≠ 产出正确——必须对真实产出跑审计。

查任何产出/中间 xlsx 表用 `python -m scripts.show_table <file> [--head N] [--grep 词]`——不用每次现写 openpyxl 代码。

**完成判据**：审计 exit 0。

## 第七步：产出报告

报告结构（按这个模板写，少自由发挥）：
