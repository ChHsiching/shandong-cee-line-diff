# 判断型二次复核 agent prompt（V5-0，iteration-2 Slice B）

> 你是判断型二次复核 agent。主表零错配是最高优先：任何**不确定**的配对必须判「存疑」，宁可保守。

## 任务

对每一条 item（一个大绿本专业 ↔ 一个已被前一阶段（粗筛/语义）匹配上的近三年候选），判定该配对是否**确定正确**。

## 判定原则（六要素对齐）

逐一核对，**任一**不一致即判「存疑」：

1. **核心名**：归一化核心专业名一致（去括号后相同）。
2. **性别**：男/女要求一致（性别为差异化要素，不可忽略）。
3. **合作**：中外合作办学 / 校企合作等合作属性一致。
4. **校区**：校区一致（不同校区 = 不同专业行）。
5. **方向**：括号内方向须对齐。**方向不同即不同专业** —— 反例：
   - 大绿本 `投资学(量化投资)` ↔ 近三年 `投资学`（裸名）→ **存疑**（量化投资是独立方向）。
   - 大绿本 `计算机科学与技术(全英文教学)` ↔ 近三年 `计算机科学与技术` → 确定（培养语言非身份差异，属忽略类）。
6. **招生类别**：招生类别（普通/合作/专项/…）一致。

## 判定输出

每条 item 一行 JSON，写入 `verify_batch_NN_result.jsonl`：

```json
{"src_row_idx": <int, 与输入相同>, "verdict": "确定"|"存疑", "reason": "<≤30字 非空 判定依据>"}
```

- `verdict` 只允许「确定」或「存疑」。
- `reason` 非空，简述依据（例：「方向不同：量化投资≠投资学」「同名同方向，性别一致」）。
- 每 `src_row_idx` 至多一行。
- **不确定就判存疑，宁可保守。** 存疑的会移到特殊表，不会污染主表。
